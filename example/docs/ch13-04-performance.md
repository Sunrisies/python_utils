## 循环与迭代器的性能对比

要判断是使用循环还是迭代器，你需要知道哪种实现更快：是带有显式`for`循环的 `search` 函数的版本，还是使用迭代器的版本。

我们执行了一个基准测试，将阿瑟·柯南·道尔所著的《夏洛克·福尔摩斯的冒险》整本书的内容加载到 `String` 中，然后查找其中的单词 _the_。以下是使用 `for` 循环和迭代器进行基准测试的结果：

```text
test bench_search_for  ... bench:  19,620,300 ns/iter (+/- 915,700)
test bench_search_iter ... bench:  19,234,900 ns/iter (+/- 657,200)
```

这两种实现在性能上非常相似！我们不会在这里解释基准测试代码，因为重点不是证明这两个版本是等效的，而是了解这两种实现在性能上的对比情况。

为了进行更全面的基准测试，你应该使用各种不同大小的文本作为 `contents`，使用不同长度的单词作为 `query`，以及所有其他可能的变体。关键在于：虽然迭代器是一种高级抽象，但编译后的代码大致与你自己编写底层代码时的代码相同。迭代器是 Rust 中一种“零成本抽象”机制，这意味着使用这种抽象不会增加额外的运行时开销。这与 Bjarne Stroustrup 在 2012 年 ETAPS 主题演讲“C++ 的基础”中定义的“零开销”概念类似。

一般来说，C++ 的实现遵循零开销原则：你不需要使用的部分，就不需要为此付出代价。此外，你确实需要使用到的部分，其实现方式应该是最优的。

在许多情况下，使用迭代器的Rust代码编译后的汇编代码与手工编写的代码相同。诸如循环展开和消除数组访问时的边界检查等优化措施会生效，从而使代码效率极高。现在你知道了这些知识，就可以放心地使用迭代器和闭包了！它们让代码看起来更高级，但并不会在运行时带来性能上的损失。

## 摘要

闭包和迭代器是受函数式编程理念启发的Rust特性。它们有助于Rust在保持低性能的同时，能够清晰地表达高级概念。闭包和迭代器的实现方式使得运行时的性能不会受到影响。这是Rust致力于提供零成本抽象的一部分目标。
