# 编写自动化测试

在1972年发表的《谦逊的程序员》一文中，埃德斯格·W·迪杰斯特拉指出：“程序测试是一种非常有效的发现错误的方法，但用它来确认程序没有错误则远远不够。”但这并不意味着我们就不应该尽可能地进行测试！

我们程序中的**正确性**是指代码能够按照我们的意图执行的程度。Rust在设计时非常注重程序的准确性，但正确性是一个复杂的问题，而且并不容易证明。Rust的类型系统承担了很大一部分责任，不过类型系统并不能涵盖所有情况。因此，Rust提供了编写自动化软件测试的支持。

假设我们编写了一个函数 `add_two`，该函数会将传入的數字加2。这个函数的签名接受一个整數作为参数，并返回一个整數作为结果。当我们实现并编译这个函数时，Rust会执行所有你之前学过的类型检查和借用检查，以确保例如我们不会传递一个 `String` 的值或无效的引用给这个函数。但是，Rust无法确保这个函数会严格按照我们的意图行事，即返回参数加2，而不是比如参数加10或参数减50！这时就需要测试来发挥作用了。

我们可以编写测试，例如，当我们将 `3` 传递给`add_two`函数时，返回的值应该是 `5`。每当我们对代码进行更改时，都可以运行这些测试，以确保现有的正确行为没有发生变化。

测试是一项复杂的技能：虽然我们无法在某一章中涵盖编写优质测试的所有细节，但在这一章中，我们将讨论Rust的测试功能机制。我们将介绍在编写测试时可用的注释和宏，测试运行的默认行为和可选选项，以及如何将测试组织为单元测试与集成测试。
