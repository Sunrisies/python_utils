## 附录E：版本信息

在第一章中，你了解到 `cargo new` 会在你的 _Cargo.toml_ 文件中添加一些关于版本的信息。这个附录将解释这些信息的含义！

Rust 语言和编译器有一个为期六周的发布周期，这意味着用户会不断获得新的功能。其他编程语言则更频繁地发布重大更新；而 Rust 则更频繁地发布小规模的更新。随着时间的推移，这些微小的更新最终会累积起来。但是，在每次发布之间，很难回过头来感慨：“哇，从 Rust 1.10 到 Rust 1.31，Rust 确实发生了很大的变化！”

大约每三年左右，Rust团队会推出一个新的Rust版本。每个版本都会将那些已经明确打包的功能整合在一起，并完全更新了相关文档和工具。新版本会通过通常的六周发布流程进行发布。

不同版本对不同的人来说有着不同的用途：

- 对于活跃的 Rust 用户来说，新版本将一系列改进整合为一个易于理解的包。
- 对于非用户来说，新版本意味着一些重大进展已经实现，这可能会让 Rust 值得再次关注。
- 对于正在开发 Rust 的人来说，新版本为整个项目提供了一个新的发展方向。

在撰写本文时，有四种Rust版本可供选择：Rust 2015、Rust 2018、Rust 2021和Rust 2024。本书采用Rust 2024版本的惯用法编写。

在_Cargo.toml_文件中，`edition`这个键表示编译器应该为你的代码使用哪个版本。如果这个键不存在，出于向后兼容性的原因，Rust会使用`2015`作为版本值。

每个项目可以选择使用除默认版本2015之外的其他版本。这些版本可能包含不兼容的更改，例如引入与代码中标识符冲突的新关键字。但是，除非您同意这些更改，否则即使您升级了所使用的Rust编译器版本，您的代码仍然可以编译通过。

所有Rust编译器版本都支持在该编译器发布之前存在的任何版本，并且它们可以将任何受支持版本的库链接在一起。版本的变化只会影响编译器对代码的解析方式。因此，如果你使用的是Rust 2015版本，而你的某个依赖项使用的是Rust 2018版本，那么你的项目仍然可以编译并使用该依赖项。相反的情况，即你的项目使用Rust 2018版本，而依赖项使用Rust 2015版本，同样也是可行的。

需要明确的是：大多数功能在所有版本中都是可用的。使用任何Rust版本的开发者将继续享受到随着新稳定版本的发布而带来的改进。然而，在某些情况下，尤其是在添加新关键字时，某些新功能可能仅适用于后续版本。如果您想利用这些功能，就需要切换版本。

如需了解更多详情，请参考[_Rust版指南_][edition-guide]。这是一本完整的书籍，详细介绍了各个版本之间的差异，并解释了如何通过 `cargo fix` 自动将代码升级到新版本。

[edition-guide]: https://doc.rust-lang.org/stable/edition-guide
