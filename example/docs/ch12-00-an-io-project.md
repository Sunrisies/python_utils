# 一个输入/输出项目：构建命令行程序

本章总结了你迄今为止学到的许多技能，并探讨了一些标准库的功能。我们将构建一个命令行工具，该工具可以处理文件以及命令行输入/输出，从而练习你现在已经掌握的一些Rust概念。

Rust 的速度、安全性、单二进制输出以及跨平台支持，使其成为创建命令行工具的理想语言。因此，在我们的项目中，我们将自己实现经典的命令行搜索工具 `grep`，该工具能够全局搜索正则表达式并输出结果。在最简单的使用场景中， `grep` 可以搜索指定文件中的特定字符串。为了实现这一功能， `grep` 需要接收两个参数：文件路径和字符串。然后，它会读取文件，找出包含该字符串的行，并输出这些行。

在这个过程中，我们将展示如何让我们的命令行工具使用许多其他命令行工具所采用的终端功能。我们将读取环境变量的值，以便用户能够配置我们的工具的行为。此外，我们还会将错误信息输出到标准错误控制台流（`stderr`），而不是标准输出流（`stdout`）。这样，例如，用户可以将成功的输出重定向到文件中，同时仍然能在屏幕上看到错误信息。

一位Rust社区成员，Andrew Gallant，已经创建了一个功能齐全且运行速度非常快的 `grep` 版本，名为 `ripgrep`。相比之下，我们的版本会相对简单一些，但这一章节会为你提供一些背景知识，帮助你理解像 `ripgrep` 这样的实际项目。
