# 最终项目：构建多线程Web服务器

这是一段漫长的旅程，但我们已经走到了这本书的终点。在这一章中，我们将再一起构建一个项目，以演示我们在最后几章中讨论的一些概念，同时回顾一些之前的学习内容。

在我们的最后一个项目中，我们将创建一个网页服务器，该服务器会显示“Hello!”，并且在网页浏览器中看起来就像图21-1所示。

这是我们构建网络服务器的计划：

1. 了解一些关于 TCP 和 HTTP 的知识。
2. 在套接字上监听 TCP 连接。
3. 解析少量的 HTTP 请求。
4. 生成合适的 HTTP 响应。
5. 通过线程池提高服务器的吞吐量。

在开始之前，我们有两点需要说明。首先，我们使用的这种方法并不是用Rust构建Web服务器的最佳方式。社区成员已经发布了许多可在[crates.io](https://crates.io/)找到的、适用于生产的库，这些库提供的Web服务器和线程池实现比我们自己所构建的更为完整。然而，本章的目的是帮助你学习，而不是走捷径。由于Rust是一种系统编程语言，我们可以选择所需的抽象级别，并可以深入到比其他语言更低的层次。

其次，我们在这里不会使用异步和等待机制。单独构建线程池就已经是一个相当大的挑战了，更何况还要再加上一个异步运行时环境！不过，我们会注意到异步和等待机制可能适用于本章中讨论的一些问题。最终，正如我们在第17章中提到的那样，许多异步运行时环境都是使用线程池来管理它们的工作的。

因此，我们将手动编写基本的 HTTP 服务器和线程池，这样你就可以了解将来可能会使用的那些库背后的基本概念和技术。
