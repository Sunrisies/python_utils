# 包、 crate 和模块

在编写大型程序时，组织代码会变得越来越重要。通过将相关的功能进行分组，并将具有不同功能的代码分开，你可以明确知道在哪里可以找到实现特定功能的代码，以及在哪里可以修改某个功能的实现方式。

到目前为止，我们编写的程序都集中在一个文件中的一个模块中。随着项目的扩展，应该将代码拆分成多个模块，然后再拆分成多个文件。一个包可以包含多个二进制模块，并且可以选择性地包含一个库模块。随着包的发展，可以将部分内容提取出来，形成独立的模块，这些模块可以作为外部依赖。本章将介绍所有这些技术。对于由多个相互关联的包组成的非常大的项目，Cargo提供了工作区功能，我们将在第十四章的[“Cargo工作区”][workspaces]<!-- ignore -->部分进行介绍。

我们还将讨论封装实现细节的方法，这样你就可以在更高层次上复用代码：一旦你实现了某个操作，其他代码就可以通过其公共接口调用你的代码，而无需了解该实现的具体实现方式。你编写代码的方式决定了哪些部分对其他代码是公开的，哪些部分则是私有的实现细节，这些细节由你保留修改的权利。这是另一种限制你需要记住的细节数量的方法。

一个相关的概念是作用域：代码编写所处的嵌套环境中有一组被定义为“在作用域内”的名称。在读取、编写和编译代码时，程序员和编译器需要知道某个特定位置的某个名称指的是变量、函数、结构体、枚举、模块、常量还是其他项，以及该项的含义。可以创建作用域，并改变哪些名称处于作用域内或不在作用域内。在同一个作用域中不能有两个具有相同名称的项；有工具可以解决名称冲突的问题。

Rust 拥有许多功能，可以帮助你管理代码的组织结构，包括哪些细节是公开的、哪些细节是私有的，以及程序中各个作用域中使用了哪些名称。这些功能有时被统称为 _模块系统_，其中包括：

* **包**: Cargo中的一个功能，允许你构建、测试和共享库或可执行文件。
* **库**: 由多个模块组成的集合，最终生成库或可执行文件。
* **模块与使用**: 允许你控制路径的组织、范围和隐私。
* **路径**: 对某个项目进行命名的方式，例如结构体、函数或模块。

在本章中，我们将介绍所有这些功能，讨论它们如何相互作用，并说明如何使用这些功能来管理作用域。通过本章的学习，你应该能够充分理解模块系统，并能够像专业人士一样处理作用域问题！
