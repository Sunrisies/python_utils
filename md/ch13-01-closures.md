## 闭包

Rust中的闭包是匿名函数，可以保存在变量中，或者作为参数传递给其他函数。可以在一个地方创建闭包，然后在其他地方调用该闭包以在不同的上下文中执行它。与函数不同，闭包可以从定义它们的作用域中捕获值。我们将演示这些闭包特性如何允许代码的重用和行为的定制。

### 推断和注释闭包类型

函数和闭包之间存在更多差异。与`fn`这样的函数不同，闭包通常不需要对参数类型或返回值进行注解。对函数进行类型注解是必要的，因为这些类型属于明确提供给用户的接口的一部分。严格定义这一接口非常重要，这样才能确保每个人都明白函数使用并返回的值的类型。而闭包则不需要这样做。在这样一个暴露的接口中：它们被存储在变量中，并且在不使用名称的情况下被使用，同时这些变量也暴露给了我们的库的用户。

闭包通常比较简短，且只在特定的上下文中才有意义，而不是在任何任意的情况下都适用。在这些有限的上下文中，编译器可以推断出参数的类型和返回类型的类型，这与它推断大多数变量类型的方式类似（不过也有极少数情况下，编译器需要闭包类型的注解）。

与变量类似，如果我们希望提高明确性和清晰度，就可以添加类型注解。不过这样做会使得代码更加冗长，除非绝对有必要。对于闭包的类型注解，其定义类似于清单13-2所示。在这个例子中，我们定义了一个闭包，并将其存储在一个变量中，而不是像清单13-1那样在传递参数的地方直接定义闭包。

### 将捕获的值移出闭包

一旦一个闭包从定义它的环境中捕获了对某个值的引用或所有权（从而影响了该值是否会被移入闭包中），那么闭包主体中的代码就会决定在之后评估闭包时，这些引用或值会发生什么变化（从而影响了这些值是否会被移出闭包）。

闭包体可以执行以下操作之一：将捕获的值移出闭包；修改捕获的值；既不移动也不修改该值；或者根本不从环境中捕获任何值。

闭包如何捕获和处理来自环境的值，会影响它实现哪些特性。而这些特性决定了函数和结构体可以使用什么样的闭包。根据闭包主体如何处理这些值，闭包会自动实现一种或多种`Fn`特性，这种实现方式是累加式的。

* `FnOnce` 适用于可以调用一次的闭包。所有闭包至少具备这一特性，因为所有闭包都可以被调用。如果一个闭包将捕获的值移出其作用域，那么它只会具备 `FnOnce` 这一特性，而不会具备其他 `Fn` 特性，因为它只能被调用一次。* `FnMut` 适用于那些不会将捕获的值移出其作用域，但可能会改变这些值的闭包。这样的闭包可以被多次调用。* ``Fn`` 适用于那些没有将捕获的值移出其作用域，也没有修改捕获的值的闭包，以及那些从环境中没有捕获任何内容的闭包。这些闭包可以多次被调用，而不会改变其外部环境，这一点在需要同时多次调用闭包的情况下非常重要。

让我们看一下在清单13-1中使用的`unwrap_or_else`方法在`Option<T>`上的定义：

```rust,ignore
impl<T> Option<T> {
    pub fn unwrap_or_else<F>(self, f: F) -> T
    where
        F: FnOnce() -> T
    {
        match self {
            Some(x) => x,
            None => f(),
        }
    }
}
```

请注意，`T`是一种通用类型，用于表示在`Option`的`Some`变体中的值的类型。而类型`T`也是`unwrap_or_else`函数的返回类型。例如，在`Option<String>`上调用`unwrap_or_else`的代码，将会得到`String`的结果。

接下来，请注意`unwrap_or_else`函数还包含一个通用的类型参数`F`。而`F`类型则是名为`f`参数的类型，该参数在我们调用`unwrap_or_else`时作为闭包被使用。

在泛型类型`F`上指定的特性绑定是`FnOnce() -> T`，这意味着`F`必须能够被调用一次，不接受任何参数，并且返回`T`。在特性绑定中使用`FnOnce`表示约束：`unwrap_or_else`不得多次调用`f`。在`unwrap_or_else`的主体中，我们可以看到，如果`Option`是`Some`的话，那么`f`就不会被调用。如果`Option`等于`None`，那么`f`将会被调用一次。因为所有闭包都实现了`FnOnce`的功能，所以`unwrap_or_else`能够接受这三种类型的闭包，并且具有最大的灵活性。

注意：如果我们想要做的事情不需要从环境中获取某个值，我们可以使用函数的名称来代替闭包，这样我们就可以使用那些实现了`Fn`特性的函数。例如，对于一个`Option<Vec<T>>`类型的值，如果该值是`None`类型，我们可以调用`unwrap_or_else(Vec::new)`来获取一个新的空向量。编译器会自动选择适用于函数定义的`Fn`特性中的任何一个来实施相应的操作。
