### 使用匹配守卫添加条件语句

匹配守卫是一种额外的`if`条件，它位于模式之后的`match`分支中，该条件也必须得到满足，才能选择该分支。匹配守卫适用于表达比单一模式所能实现的更复杂的逻辑。不过需要注意的是，它们仅适用于`match`表达式，而不适用于`if let`或`while let`表达式。

该条件可以使用在模式中创建的变量。清单19-26展示了一个`match`结构，其中第一个分支使用了模式`Some(x)`，并且还有一个匹配守卫`if x % 2 == 0`（如果数字是偶数，则变为`true`）。

```rust

```

</清单>

这个示例将打印`The number 4 is even`。当`num`与第一个分支中的模式匹配时，匹配成功是因为`Some(4)`与`Some(x)`相匹配。然后，匹配判断会检查`x`除以2的余数是否等于0，如果等于0，则选择第一个分支。

如果`num`实际上是`Some(5)`，那么第一个分支中的匹配守卫就会是`false`，因为5除以2的余数是1，不等于0。在这种情况下，Rust会执行第二个分支，而第二个分支会进行匹配，因为第二个分支没有匹配守卫，所以它可以匹配任何`Some`的变体。

无法在模式中直接表达`if x % 2 == 0`条件，因此，匹配守卫为我们提供了表达这种逻辑的能力。不过，这种额外表达方式的缺点是，当涉及到匹配守卫表达式时，编译器不会尝试检查其完备性。

在讨论清单19-11时，我们提到可以使用匹配守卫来解决模式遮蔽问题。回想一下，我们在`match`表达式中创建了一个新的变量，而不是使用位于`match`之外的变量。这个新的变量意味着我们无法测试外部变量的值。清单19-27展示了如何使用匹配守卫来解决这个问题。

### 使用 `@` 绑定

`at`运算符 ``@`` 让我们能够创建一个变量，在测试某个值的模式匹配时，同时存储该值。在清单19-29中，我们想要测试一个 ``Message::Hello`` 到 ``id`` 的字段是否在 ``3..=7`` 的范围内。我们还希望将该值绑定到变量 ``id`` 上，这样我们就可以在与之相关的代码中使用该值。

使用 ``@`` 可以让我们测试一个值，并将其保存在一个变量中，而只需在一个模式中完成操作。

## 摘要

Rust中的模式在区分不同类型的数据方面非常有用。当在`match`表达式中使用时，Rust确保你的模式能够覆盖所有可能的值，否则程序将无法编译。在`let`语句和函数参数中使用的模式使得这些结构更加实用，能够将值分解为更小的部分，并将这些部分分配给变量。我们可以创建简单或复杂的模式来满足我们的需求。
