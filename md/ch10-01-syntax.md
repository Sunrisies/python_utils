## 通用数据类型

我们使用泛型来为函数签名或结构体等元素定义规范，然后可以将这些定义应用于多种具体数据类型。首先，让我们看看如何使用泛型来定义函数、结构体、枚举类型和方法。接着，我们将讨论泛型对代码性能的影响。

### 在函数定义中

在定义使用泛型的函数时，我们将泛型放在函数的签名中，就像通常指定参数和返回值的数据类型一样。这样做可以使我们的代码更加灵活，并为函数的调用者提供更多的功能，同时还能避免代码重复。

要在一个新的函数中为类型参数化，我们需要为类型参数命名，就像为函数的值参数命名一样。你可以使用任何标识符作为类型参数的名称。不过，我们会使用`T`这个名称，因为按照惯例，Rust中的类型参数名称通常很短，往往只有一个字母，而且Rust的类型命名 conventions是UpperCamelCase。`T`是大多数Rust程序员默认的选择。

当我们在函数体内使用参数时，必须在函数签名中声明该参数的名称，这样编译器才能理解这个名称的含义。同样地，当我们在函数签名中使用类型参数名称时，也必须在使用之前先声明该类型参数名称。为了定义泛型`largest`函数，我们需要在尖括号内声明类型名称，即`<>`，并将其放在函数名称和参数列表之间。

```rust,ignore
fn largest<T>(list: &[T]) -> &T {
```

我们将这个定义翻译为：“函数`largest`是某种类型的通用函数，其参数名为`list`，该参数是一个类型为`T`的值的切片。函数`largest`将返回一个指向相同类型`T`的值的引用。”

### 在枚举定义中

就像我们处理结构体一样，我们也可以定义枚举类型来保存各种数据类型的变体。让我们再看一下标准库提供的`Option<T>`枚举类型，我们在第6章中曾经使用过它。

```rust
enum Option<T> {
    Some(T),
    None,
}
```

现在，这个定义应该对你来说更有意义了。如你所见，``Option<T>``枚举类型比``T``更通用，它有两个变体：``Some``，它包含一个类型为``T``的值；还有一个``None``变体，它不持有任何值。通过使用``Option<T>``枚举，我们可以表达一个可选值的抽象概念。由于``Option<T>``是通用的，我们就可以使用这种抽象概念。无论可选值的类型是什么。

枚举类型也可以使用多种泛型类型。我们在第9章中使用的`Result`枚举就是一个例子：

```rust
enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

`Result`枚举类型包含两个子类：`T`和`E`。该枚举有两个变体：`Ok`，其值为`T`类型；以及`Err`，其值为`E`类型。这样的定义使得我们可以在任何可能成功执行操作（返回某种`T`类型的数值）或可能失败的操作中，方便地使用`Result`枚举类型。（返回某种类型的错误 `E`）。实际上，这就是我们在清单9-3中用来打开文件的方法，其中 `T` 在文件成功打开时被填充为类型 `std::fs::File`，而在文件打开出现问题时，`E` 则被填充为类型 `std::io::Error`。

当你在代码中发现多个结构体或枚举定义，它们只是持有的值的类型不同时，可以通过使用泛型类型来避免重复。

### 使用泛型提高代码性能

您可能在想，使用泛型类型参数时是否会带来运行时的开销。好消息是，使用泛型类型并不会使程序的运行速度比使用具体类型时更慢。

Rust通过在编译时使用泛型对代码进行单态化来实现这一点。所谓“单态化”，就是通过将泛型代码中使用的具体类型填充进来，将泛型代码转化为具体代码的过程。在这个过程中，编译器所做的与我们在清单10-5中创建泛型函数时所使用的步骤相反：编译器会检查所有调用泛型代码的地点，并为这些泛型代码所依赖的具体类型生成相应的代码。

让我们看看如何使用标准库的通用`Option<T>`枚举来实现这一功能。

```rust
let integer = Some(5);
let float = Some(5.0);
```

当Rust编译这段代码时，会执行单态化操作。在这个过程中，编译器会读取在`Option<T>`中使用的变量值，并识别出两种类型的`Option<T>`：一种就是`i32`，另一种则是`f64`。因此，编译器会将`Option<T>`的泛型定义扩展为两个针对`i32`和`f64`的特殊化定义，从而用这些具体的定义替换了原来的泛型定义。

该代码的单态化版本看起来类似于下面这个例子（编译器使用的名称与我们这里用来说明的有所不同）：

<listing file-name="src/main.rs">

```rust
enum Option_i32 {
    Some(i32),
    None,
}

enum Option_f64 {
    Some(f64),
    None,
}

fn main() {
    let integer = Option_i32::Some(5);
    let float = Option_f64::Some(5.0);
}
```

</清单>

通用类型 ``Option<T>`` 被编译器创建的具体定义所替代。由于Rust将泛型代码编译成每个实例都指定了类型的代码，因此使用泛型在运行时不会产生任何成本。当代码运行时，它的执行方式与我们手动复制每个定义的方式完全相同。这种单态化过程使得Rust的泛型在运行时非常高效。
