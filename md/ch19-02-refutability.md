## 可反驳性：某个模式是否可能无法匹配

模式有两种形式：可反驳的和不可反驳的。那些对于任意给定的值都能匹配的模式被称为“不可反驳的”。例如，``x``在``let x = 5;``中，因为``x``可以匹配任何值，因此不可能不匹配。而那些在某些特定值下可能会不匹配的模式则被称为“可反驳的”。例如，``Some(x)``在表达式`if let Some(x) = a_value` because if the value in the `a_value` variable is `None`中，就可能不会匹配。`Some`的模式与`Some(x)`的模式不匹配。

函数参数、`let`语句以及`for`循环只能接受可反驳的模式，因为当值不匹配时，程序无法执行任何有意义的操作。`if let`和`while let`表达式以及`let...else`语句可以接受可反驳和不可反驳的模式，但编译器会警告不可反驳的模式，因为从定义上来说，它们旨在处理可能出现的故障：条件语句的功能在于……它的表现会根据成功或失败的不同情况而发生变化。

一般来说，您不必担心可反驳模式与不可反驳模式之间的区别。不过，您需要熟悉可反驳性的概念，这样在看到错误信息时就能做出相应的回应。在这种情况下，根据代码的预期行为，您需要修改所使用的模式或与之结合使用的构造方式。
