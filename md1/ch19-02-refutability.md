## 可反驳性：某个模式是否可能无法匹配

模式有两种形式：可反驳的和不可反驳的。那些对于任意给定的值都能匹配的模式被称为“不可反驳的”。例如，在语句 ``let x = 5;`` 中，``x`` 可以匹配任何值，因此必然能够匹配。而那些在某些特定情况下可能无法匹配的模式则被称为“可反驳的”。例如，在表达式 `if let Some(x) = a_value` because if the value in the `a_value` variable is `None` 中，``Some(x)`` 可能无法匹配 ``Some``，因为 ``Some(x)`` 无法匹配该模式。

函数参数、`let`语句以及`for`循环只能接受不可辩驳的模式，因为当值不匹配时，程序无法执行任何有意义的操作。而`if let`和`while let`表达式以及`let...else`语句则可以接受可辩驳和不可辩驳的模式，但编译器会警告不要使用不可辩驳的模式，因为从定义上来说，这些模式就是为了处理可能出现的失败情况：条件语句的功能在于根据成功或失败的状况来做出不同的行为。

一般来说，你不需要担心可反驳与不可反驳的模式之间的区别；不过，你需要熟悉可反驳性的概念，这样当你在错误信息中看到这种模式时，就能做出相应的回应。在这种情况下，你需要根据代码的预期行为，改变使用该模式的构造方式或模式所关联的构造方式。
