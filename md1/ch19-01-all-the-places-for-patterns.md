## 所有地方都可以使用这种模式

在Rust中，模式出现在许多地方，而你可能一直在不知不觉中使用它们！本节将讨论所有允许使用模式的场景。

### `match` 武器

如第6章所述，我们在`match`表达式中使用模式。  
正式来说，`match`表达式被定义为关键字`match`、一个用于匹配的值，以及一个或多个匹配臂，这些匹配臂由一个模式和一个表达式组成，当该值与该臂的模式匹配时，就会执行这个表达式。

  <pre><code>match <em>VALUE</em> {

例如，以下是清单6-5中的`match`表达式，该表达式会在变量`x`中查找与`Option<i32>`值相匹配的内容。

```rust,ignore
match x {
    None => None,
    Some(i) => Some(i + 1),
}
```

这个`match`表达式中的模式，就是每个箭头左侧的`None`和`Some(i)`。

`match`表达式的一个要求是，它们必须涵盖所有可能性，也就是说，`match`表达式中可能的值的所有情况都必须被考虑进去。为了确保涵盖了所有可能性，可以在最后一个分支中使用一个通用的模式。例如，一个与任何值都匹配的变量名永远不会导致错误，因此可以覆盖所有剩余的情况。

特定的模式`_`可以匹配任何内容，但它永远不会绑定到某个变量，因此它通常用在最后一个匹配项中。例如，当你想要忽略未指定的值时，`_`模式就非常有用。我们将在本章的[“在模式中忽略值”][ignoring-values-in-a-pattern]<!-- ignore -->部分更详细地介绍`_$模式$。

### `let` 语句

在本章之前，我们仅明确讨论了如何在`match`和`if let`中使用模式，但实际上，我们在其他地方也使用了模式，包括在`let`语句中。例如，考虑一下使用`let`进行的简单变量赋值：

```rust
let x = 5;
```

每次你使用这样的 ``let`` 语句时，实际上你都在使用某种模式，虽然你可能没有意识到这一点！更正式地说，``let`` 语句的样子就像这样：

<!--
  手动格式化而非使用Markdown格式：Markdown并不支持在块级内容中这样对代码进行斜体处理！
-->

<pre>
<code>let PATTERN = EXPRESSION;</code>
</pre>

在像 `let x = 5;` 这样的语句中，当变量名称位于 PATTERN 位置时，该变量名称实际上是一种特别简单的模式。Rust 会将表达式与这个模式进行比较，并分配任何匹配到的名称。因此，在 `let x = 5;` 示例中，`x` 是一个模式，表示“将在这里匹配到的内容绑定到变量 `x`”。由于 `x` 是整个模式，所以这个模式实际上意味着“将所有内容绑定到变量 `x` 上，无论其值是什么。”

### 条件性 `if let` 表达式

在第六章中，我们讨论了如何使用`if let`表达式，它主要作为一种更简洁的方式来编写与`match`等效的代码，后者仅适用于一种情况。此外，`if let`还可以包含一个对应的`else`代码，当`if let`中的模式不匹配时，该代码会被执行。

清单19-3显示，也可以混合使用``if let``、``else if`, and `else if let`表达式。这样做比使用``match``表达式更加灵活，因为后者只能表达一个值与模式进行比对。此外，Rust并不要求一系列`if let`, `else if`, and `else if let`中的条件之间具有相互关联关系。

如果用户指定了喜欢的颜色，那么该颜色将被用作背景色。  
如果没有指定喜欢的颜色，并且今天是星期二，那么背景色将是绿色。否则，如果用户以字符串形式输入了自己的年龄，并且我们能够成功将其解析为数字，那么背景色将取决于该数字的值，可能是紫色或橙色。如果这些条件都不适用，那么背景色将是蓝色。

这种条件结构使我们能够支持复杂的需求。使用这里硬编码的值，这个例子将会打印出“使用紫色作为背景颜色”。

可以看到，``if let``同样可以引入新的变量，这些新变量会以与``match``的方式遮蔽现有的变量。例如，``if let Ok(age) = age``会引入一个新的``age``变量，该变量包含``Ok``变体中的值，从而遮蔽现有的``age``变量。这意味着我们需要把`if age > 30` condition within that block: We can’t combine these two conditions into `if let Ok(age) = age && age > 30`. The new `age`这一代码放在新的作用域中，因为在此之前，比较条件`30`是无效的。

使用 ``if let`` 表达式的缺点是，编译器不会进行穷举性检查，而使用 ``match`` 表达式时，编译器会进行这种检查。如果我们省略了最后一个 ``else`` 块，从而错过了一些情况的处理，那么编译器就不会提醒我们可能存在逻辑错误。
