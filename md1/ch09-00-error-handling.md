# 错误处理

在软件开发中，错误是不可避免的现象。因此，Rust提供了许多功能来处理可能出现的问题。在许多情况下，Rust要求你在代码编译之前就意识到可能存在错误，并采取相应的措施。这一要求使得程序更加健壮，因为它能确保你在将代码部署到生产环境之前发现错误并妥善处理它们！

Rust将错误分为两大类：可恢复的错误和不可恢复的错误。对于可恢复的错误，比如“文件未找到”的错误，我们通常只需要向用户报告问题并重新尝试操作即可。而不可恢复的错误则通常是程序出错的症状，例如试图访问数组之外的位置，这种情况下我们需要立即终止程序。

大多数编程语言并不区分这两种错误，而是用类似异常的机制来同时处理这两种情况。Rust没有异常机制。相反，它有一个名为`Result<T, E>`的类型，用于可恢复的错误处理；还有一个`panic!`宏，当程序遇到无法恢复的错误时，它会终止程序的执行。本章首先介绍如何调用`panic!`，然后讨论如何返回`Result<T, E>`类型的值。此外，我们还将探讨在决定是尝试从错误中恢复还是停止执行时的各种考虑因素。
