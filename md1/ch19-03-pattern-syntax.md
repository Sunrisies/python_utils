### 匹配命名变量

命名变量是一种不可辩驳的模式，可以匹配任何值，我们在本书中多次使用了它们。然而，在 ``match``、``if let`` 或 ``while let`` 表达式中使用命名变量时，会出现一个复杂情况。因为每种这样的表达式都会创建一个新的作用域，所以在这些表达式内部作为模式声明的变量会遮蔽那些在外部结构中使用相同名称的变量，就像所有变量一样。在 Listing 19-11 中，我们声明了一个名为 ``x`` 的变量，其值为 ``Some(5)``；同时声明了另一个名为 ``y`` 的变量，其值为 ``10``。然后，我们在值 ``x`` 的基础上创建了一个 ``match`` 表达式。请观察 `match` 分支中的模式以及最后的 ``println!``，试着在运行这段代码或继续阅读之前，弄清楚代码将会输出什么内容。

在第二个匹配分支中，引入了一个新的变量 ``y``，该变量可以匹配 ``Some`` 内部的值。由于我们处于 ``match`` 表达式内的新作用域中，因此这是一个新的 ``y`` 变量，而不是我们在开头用 ``10`` 声明的 ``y`` 变量。这个新的 ``y`` 变量可以匹配 ``Some`` 内部的所有值，而 ``Some`` 正是我们在 ``x`` 中看到的。因此，这个新的 ``y`` 变量会绑定到 ``Some`` 的内部值，即 ``x`` 中的值。那个值就是 ``5`$，所以该分支的表达式会执行并输出 ``Matched, y = 5`$。

如果 `x` 是一个 `None` 类型的值，而不是 `Some(5)` 类型的值，那么前两个分支中的模式就不会匹配，因此该值会匹配到下划线符号。我们在下划线分支的模式中没有引入 `x` 变量，所以表达式中的 `x` 仍然是未被遮蔽的外部 `x`。在这种假设的情况下，`match` 将会输出 `默认情况，x = None`。

当 `match` 表达式执行完毕后，其作用域也随之结束，而内部 `y` 的作用域同样终止。最后一个 `println!` 会生成 `at the end: x = Some(5), y = 10`。

为了创建一个`match`表达式，该表达式可以比较外部`x`和`y`的值，而不是引入一个新的变量来覆盖现有的`y`变量，我们需要使用匹配守卫条件语句。我们将在[“使用匹配守卫添加条件语句”](#adding-conditionals-with-match-guards)部分的后续章节中讨论匹配守卫。

### 使用 Match Guards 添加条件语句

**匹配守卫**是一种额外的`__INLINE_CODE__224__`条件，它被放在`__INLINE_CODE__225__`分支中的模式之后，该条件必须匹配才能选择该分支。匹配守卫适用于表达比单个模式所能容纳的更复杂的逻辑。不过需要注意的是，它们仅适用于`__INLINE_CODE__226__`表达式，而不适用于`__INLINE_CODE__227__`或`__INLINE_CODE__228__`表达式。

该条件可以使用在模式中创建的变量。清单19-26展示了一个`match`，其中第一个分支使用了模式`Some(x)`，并且还有一个匹配守卫`if x % 2 == 0`（如果数字是偶数，那么这个守卫的值将会是`true`）。


</ Listing>

这个示例将输出 `The number 4 is even`。当 `num` 与第一个分支中的模式进行匹配时，匹配成功，因为 `Some(4)` 与 `Some(x)` 相匹配。接着，匹配守卫会检查 `x` 除以 2 后的余数是否等于 0；如果等于 0，那么第一个分支就会被选中。

如果 `num` 被替换为 `Some(5)`，那么第一个分支中的匹配守卫就会变成 `false`。因为5除以2的余数是1，而1不等于0。此时Rust会执行第二个分支，因为第二个分支没有匹配守卫，所以它会匹配任何 `Some` 的变体。

无法在模式中直接表达 `if x % 2 == 0` 这个条件，因此，匹配守卫为我们提供了表达这种逻辑的能力。不过，这种额外表达方式的缺点是，当涉及到匹配守卫表达式时，编译器不会尝试进行穷举检查。

在讨论清单19-11时，我们提到可以使用匹配守卫来解决模式遮蔽问题。回想一下，我们在`match`表达式中为模式内部创建了一个新变量，而不是使用`match`外部的变量。这个新变量意味着我们无法测试外部变量的值。清单19-27展示了如何使用匹配守卫来解决这个问题。

### 使用 `@` 进行绑定

`_at_`运算符 ``@`` 让我们能够创建一个变量，该变量在测试某个值以进行模式匹配的同时，也保存了该值。在 Listing 19-29 中，我们希望测试 ``Message::Hello`` 和 ``id`` 字段是否在 ``3..=7`` 的范围内。我们还希望将该值绑定到变量 ``id`` 上，这样我们就可以在与该分支相关的代码中使用该变量了。

使用 ``@`` 可以让我们在一个模式中测试一个值，并将其保存在一个变量中。

## 总结

Rust中的模式在区分不同类型的数据方面非常有用。当在`match`表达式中使用时，Rust会确保你的模式能够覆盖所有可能的值，否则程序将无法编译。在`let`语句和函数参数中使用的模式使得这些构造更加实用，能够将值分解为更小的部分，并将这些部分分配给变量。我们可以创建简单或复杂的模式来满足我们的需求。
