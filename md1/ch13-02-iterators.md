## 使用迭代器处理一系列项目

迭代器模式允许你依次对一系列元素执行某些操作。迭代器负责遍历每个元素的逻辑，并判断序列是否已完成。使用迭代器时，你不必自己重新实现这一逻辑。

在Rust中，迭代器是“惰性”的，这意味着在你调用消耗该迭代器的方法之前，迭代器并不会产生任何效果。例如，清单13-10中的代码通过调用定义在`Vec<T>`上的`iter`方法，来创建一个针对向量`v1`中元素的迭代器。这段代码本身并不会执行任何有用的操作。

在那些标准库中没有提供迭代器的语言中，你可能需要通过以索引0开始一个变量，然后使用该变量来索引向量以获取值，并在循环中不断增加这个变量的值，直到达到向量中项目的总数。

迭代器会处理所有这些逻辑，从而减少了可能出错的重复代码。迭代器让你可以更灵活地使用相同的逻辑来处理多种不同类型的序列，而不只是像向量那样可以通过索引访问的数据结构。让我们来看看迭代器是如何实现这一点的。

### `Iterator` 特质与 `next` 方法

所有迭代器都实现了一种名为 ``Iterator`` 的特质，该特质定义在标准库中。这个特质的定义如下所示：

```rust
pub trait Iterator {
    type Item;

    fn next(&mut self) -> Option<Self::Item>;

    // methods with default implementations elided
}
```

请注意，这个定义使用了一些新的语法：`type Item` 和 `Self::Item`，它们用于定义一个具有该特性的关联类型。我们将在第20章中详细讨论关联类型。目前，你需要知道的是，这段代码表明要实现 `Iterator` 特性，就必须同时定义 `Item` 类型，而 `Item` 类型则被用于 `next` 方法的返回类型中。换句话说，`Item` 类型将是从迭代器中返回的类型。

`Iterator` 特质要求实现者仅定义一个方法：即 `next` 方法。该方法每次返回一个迭代器中的元素，该元素会被包装在 `Some` 中。当迭代结束后，该方法会返回 `None`。

我们可以直接在迭代器上调用`next`方法；清单13-12展示了在从向量创建的迭代器上重复调用`next`时返回的值。

请注意，我们需要将 ``v1_iter`` 设为可变的：在迭代器上调用 ``next`` 方法会更改迭代器用于跟踪其在序列中位置的内部状态。换句话说，这段代码会“消耗”掉迭代器。每次调用 ``next`` 都会从迭代器中获取一个元素。当我们使用 ``for`` 循环时，不需要将 ``v1_iter`` 设为可变，因为循环会接管 ``v1_iter`` 的所有权，并在后台将其变为可变状态。

另外需要注意的是，我们从`next`调用中得到的数值都是对向量中值的不可变引用。而`iter`方法则会产生一个基于这些不可变引用的迭代器。如果我们想要创建一个能够获取`v1`中的值并返回这些已占用的资源的迭代器，我们可以调用`into_iter`而不是`iter`。同样地，如果我们想要遍历可变引用，我们可以调用`iter_mut`而不是`iter`。

### 消耗迭代器的方法

`Iterator`特质包含多种方法，这些方法由标准库提供默认实现；您可以通过查阅标准库中关于`Iterator`特质的API文档来了解这些方法。其中一些方法在其定义中调用了`next`方法，这就是为什么在实现`Iterator`特质时，必须实现`next`方法的原因。

那些调用 ``next`` 的方法被称为“消耗型适配器”，因为调用它们会占用迭代器的资源。一个例子就是 ``sum`` 方法，该方法会占用迭代器的所有权，并通过反复调用 ``next`` 来遍历其中的元素，从而消耗掉迭代器。在遍历过程中，它会将每个元素累加到总和中，并在遍历完成后返回总计值。清单13-13展示了一个使用 ``sum`` 方法的示例。

### 捕获其环境的闭包

许多迭代器适配器会将闭包作为参数传递，而通常我们指定给迭代器适配器的闭包都是能够捕获其环境的闭包。

在这个示例中，我们将使用`filter`方法，该方法接受一个闭包作为参数。这个闭包从迭代器中获取一个元素，并返回一个`bool`。如果闭包返回`true`，那么这个值将被包含在由`filter`产生的迭代结果中。如果闭包返回`false`，那么这个值则不会被包含在内。

`shoes_in_size`函数接受一个鞋类向量和一个鞋码作为参数。它返回一个只包含指定尺码鞋子的向量。

在 `shoes_in_size` 的主体中，我们调用 `into_iter` 来创建一个迭代器，该迭代器拥有该向量。然后，我们调用 `filter` 来将该迭代器适配成一个新的迭代器，这个新迭代器只包含那些使得闭包返回 `true` 的元素。

该闭包从环境中获取`shoe_size`参数，并将其与每只鞋子的尺寸进行比较，只保留符合指定尺寸的鞋子。最后，调用`collect`将适应器迭代器返回的值收集到一个向量中，该向量由该函数返回。

测试表明，当我们调用`shoes_in_size`时，返回来的只有与我们指定的尺寸相同的鞋子。
