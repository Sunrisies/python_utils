## 将智能指针视为普通引用

实现 ``Deref`` 特性可以让你自定义 `_dereference` 运算符 ``*`` 的行为（不要与乘法或全局运算符混淆）。通过实现 ``Deref``，使得智能指针可以被当作普通的引用来处理，这样你就可以编写操作引用的代码，并且将这些代码与智能指针一起使用。

首先，让我们看看解引用运算符是如何与常规引用协同工作的。接着，我们将尝试定义一个自定义类型，该类型的行为类似于`Box<T>`，并了解为什么解引用运算符在我们对新定义的类型上并不像引用那样工作。我们将探讨如何实现`Deref`特性，从而使智能指针能够以类似引用的方式使用。最后，我们将了解Rust的解引用强制转换功能，以及它如何让我们能够同时使用引用和智能指针。

### 实现 `Deref` 特质

如第10章中的[“在类型上实现特质”][impl-trait]所述，要实现一个特质，我们需要为该特质所需的方法提供实现。标准库提供的`Deref`特质要求我们实现一个名为`deref`的方法，该方法会借用`self`并返回对内部数据的引用。清单15-10展示了`Deref`的实现，该实现被添加到`MyBox<T>`的定义中。

```rust
{{#rustdoc_include ../listings/ch15-smart-pointers/listing-15-10/src/main.rs:here}}
```

</ Listing>

`type Target = T;`语法为`Deref`特质定义了一个关联类型，以便使用。关联类型是一种声明泛型参数的不同方式，但目前不必担心它们；我们将在第20章中更详细地介绍它们。

我们将 `&self.0` 填充到 `deref` 的方法体中，这样 `deref` 就能返回对我们使用 `*` 运算符所要访问的值的引用；回想一下在第五章的“使用元组结构体创建不同类型”部分中提到的内容，即 `.0` 可以访问元组结构体中的第一个值。清单 15-9 中的 `main` 函数调用了 `*` 对 `MyBox<T>` 值的操作，现在代码可以编译了，而且所有的断言也都通过了！

如果没有`Deref`特性，编译器只能解引用`&`中的引用。而`deref`方法则使编译器能够获取任何实现了`Deref`的类型的值，并调用`deref`方法来获取一个可以解引用的引用。

当我们执行到清单15-9中的`*y`时，Rust实际上在后台运行了这段代码：

```rust,ignore
*(y.deref())
```

Rust 将 `*` 运算符替换为对 `deref` 方法的调用，然后是一个简单的解引用操作，这样我们就无需考虑是否需要调用 `deref` 方法了。这一特性使得我们可以编写出无论使用常规引用还是实现了 `Deref` 类型的类型，都能正常工作的代码。

之所以 ``deref`` 方法返回对某个值的引用，而 ``*(y.deref())`` 中外部的直接解引用操作仍然是必要的，这与所有权系统有关。如果 ``deref`` 方法直接返回该值，而不是返回对该值的引用，那么该值就会从 ``self`` 中被移出。在这种情况下，或者在我们使用解引用操作符的大多数情况下，我们都不希望拥有 ``MyBox<T>`` 内部的值的所有权。

请注意，`*`运算符被替换为对`deref`方法的调用，然后再次调用`*`运算符，每次调用时都会使用`*`。由于`*`运算符的替换不会无限递归，最终我们得到的是类型为`i32`的数据，这些数据与清单15-9中的`assert_eq!`中的`5`相匹配。

### 在函数和方法中使用去引用强制类型

**解引用强制转换**将指向实现了``Deref``特性的类型的引用，转换为指向另一个类型的引用。例如，解引用强制转换可以将``&String``转换为``&str``，因为``String``实现了``Deref``特性，从而返回``&str``。解引用强制转换是Rust对函数和方法参数进行的一种便利操作，它仅适用于实现了``Deref``特性的类型。当我们将指向特定类型值的引用作为参数传递给一个与函数或方法定义中的参数类型不匹配的函数或方法时，这种转换会自动发生。对``deref``方法的连续调用会将我们提供的类型转换为参数所需的类型。

Rust中增加了解引用强制功能，这样编写函数和方法的程序员就不需要使用``&``和``*``来手动添加和删除引用了。这种解引用强制功能还允许我们编写适用于引用或智能指针的代码。

### 使用可变引用处理反向引用强制转换

类似于使用 ``Deref`` 特性来覆盖不可变引用上的 ``*`` 运算符，您也可以使用 ``DerefMut`` 特性来覆盖可变引用上的 ``*`` 运算符。

当在三种情况下发现类型以及特性实现时，Rust会执行去引用强制转换操作：

1. 从 `&T` 到 `&U`，在 `T: Deref<Target=U>` 时  
2. 从 `&mut T` 到 `&mut U`，在 `T: DerefMut<Target=U>` 时  
3. 从 `&mut T` 到 `&U`，在 `T: Deref<Target=U>` 时

前两种情况是相同的，只是第二种情况实现了可变性。  
第一种情况指出，如果你有一个 ``&T``，而 ``T`` 实现了 ``Deref`` 到某种类型 ``U``，那么你可以透明地获取 ``&U``。第二种情况则指出，对于可变引用也会发生相同的解引用操作。

第三种情况更为复杂：Rust还会将可变引用强制转换为不可变引用。但是反过来是不可能的：不可变引用永远无法被强制转换为可变引用。由于借用规则的存在，如果你有一个可变引用，那么这个可变引用必须是对该数据的唯一引用（否则程序将无法编译）。将一个可变引用转换为不可变引用不会违反借用规则。然而，将不可变引用转换为可变引用则需要初始的不可变引用是该数据的唯一不可变引用，但借用规则并不保证这一点。因此，Rust无法假设将不可变引用转换为可变引用是可行的。
